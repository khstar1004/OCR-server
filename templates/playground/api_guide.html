army-ocr API Guide

이 문서는 외부 서비스가 army-ocr을 호출해 PDF/이미지를 구조화 문서로 바꾸는 방법을 설명합니다. 구현된 OCR 서비스 API만 다루며, 브라우저 체험 UI 전용 endpoint는 마지막 보조 항목으로만 분리합니다.

연동 기준 제품 연동은 /api/v1/*를 기준으로 합니다. /playground/*는 사람이 결과를 확인하는 데모/QA 클라이언트입니다.

Quickstart

가장 단순한 사용법은 파일을 /api/v1/marker에 올리고, request_id로 결과를 조회하는 흐름입니다.

BASE_URL="http://127.0.0.1:18110"

curl "$BASE_URL/health"
curl "$BASE_URL/api/v1/capabilities"

curl -X POST "$BASE_URL/api/v1/marker" \
  -F "file=@sample.pdf" \
  -F "output_format=json,markdown,html,chunks" \
  -F "page_range=0" \
  -F "mode=balanced"

curl "$BASE_URL/api/v1/marker/{request_id}"

Python 예시

import time
import requests

base_url = "http://127.0.0.1:18110"

with open("sample.pdf", "rb") as file:
    submit = requests.post(
        f"{base_url}/api/v1/marker",
        files={"file": file},
        data={
            "output_format": "json,markdown,html,chunks",
            "page_range": "0",
            "mode": "balanced",
        },
    ).json()

request_id = submit["request_id"]
while True:
    result = requests.get(f"{base_url}/api/v1/marker/{request_id}").json()
    if result["status"] != "processing":
        break
    time.sleep(1)

print(result["status"])
print(result.get("markdown", ""))

Direct OCR

Direct OCR는 한 번의 응답에서 OCR 결과를 바로 받습니다. 짧은 이미지/PDF 테스트에 적합합니다.

Endpoint입력결과
POST /api/v1/ocr/image이미지 파일text, markdown, blocks, bbox
POST /api/v1/ocr/pdfPDF 파일pages 배열, 페이지별 blocks
 
curl -X POST "$BASE_URL/api/v1/ocr/image" \
  -F "file=@page.png" \
  -F "include_raw=false"

curl -X POST "$BASE_URL/api/v1/ocr/pdf" \
  -F "file=@sample.pdf" \
  -F "page_range=0-2" \
  -F "include_raw=false"

Document Convert

/api/v1/marker는 OCR 결과를 JSON, Markdown, HTML, chunks로 구조화합니다. 일반 연동에서는 이 endpoint를 우선 사용합니다.

Field기본값설명
file 또는 file.0-업로드 파일
file_url-HTTP(S), file path, file URL
output_formatmarkdownjson,markdown,html,chunks 복수 지정
modebalancedfast, balanced, accurate
page_range전체zero-based. 예: 0, 0-2, 0,2
max_pages전체처리할 최대 페이지 수
paginatefalse페이지 구분 유지
add_block_idsfalseHTML/블록에 ID 추가
include_markdown_in_chunksfalsechunk에 markdown 포함

결과 주요 필드

{
  "status": "complete",
  "success": true,
  "request_id": "req_...",
  "json": {

        "height": 900,
        "blocks": [
          {"label": "text", "bbox": [118, 130, 886, 194], "text": "..."}
        ]
      }
    ]
  },
  "markdown": "...",

Files

파일을 먼저 저장해두면 문서 생성, 변환, 분할, 구조화 추출, batch run에서 같은 파일을 재사용할 수 있습니다.

curl -X POST "$BASE_URL/api/v1/files" \
  -F "file=@sample.pdf"

curl "$BASE_URL/api/v1/files"
curl "$BASE_URL/api/v1/files/{file_id}/metadata"
curl "$BASE_URL/api/v1/files/{file_id}/download"

Upload slot 방식

curl -X POST "$BASE_URL/api/v1/files/request_upload_url" \
  -H "content-type: application/json" \
  -d "{\"file_name\":\"sample.pdf\",\"content_type\":\"application/pdf\"}"

curl -X PUT "$BASE_URL/api/v1/files/uploads/{upload_id}" \
  --data-binary "@sample.pdf"

curl "$BASE_URL/api/v1/files/uploads/{upload_id}/confirm"

Documents

Document API는 파일 record를 문서 단위로 관리하고, 변환/분할/추출 흐름의 기준 ID를 제공합니다.

  1. 파일 등록POST /api/v1/files로 file_id를 받습니다.
  2. 문서 생성POST /api/v1/create_document로 document_id를 만듭니다.
  3. 문서 변환POST /api/v1/convert_document로 JSON/Markdown/HTML/chunks를 생성합니다.
  4. 문서 분할POST /api/v1/segment_document로 기사/섹션 단위 결과를 만듭니다.
curl -X POST "$BASE_URL/api/v1/create_document" \
  -H "content-type: application/json" \
  -d "{\"file_id\":\"file_...\"}"

curl "$BASE_URL/api/v1/create_document/{request_id}"

curl -X POST "$BASE_URL/api/v1/convert_document" \
  -H "content-type: application/json" \
  -d "{\"document_id\":\"doc_...\",\"output_format\":\"json,markdown\"}"

curl -X POST "$BASE_URL/api/v1/segment_document" \
  -H "content-type: application/json" \
  -d "{\"document_id\":\"doc_...\",\"page_range\":\"0\"}"

Extraction

구조화 추출은 OCR 텍스트와 문서 분할 결과를 기반으로 필드 값을 찾습니다. 현재 구현은 rule-first 방식입니다.

curl -X POST "$BASE_URL/api/v1/generate_extraction_schemas" \
  -H "content-type: application/json" \
  -d "{\"field_names\":[\"title\",\"summary\",\"document_date\"]}"

curl "$BASE_URL/api/v1/generate_extraction_schemas/{request_id}"

curl -X POST "$BASE_URL/api/v1/extract_structured_data" \
  -H "content-type: application/json" \
  -d "{\"document_id\":\"doc_...\",\"schema\":{\"name\":\"report\",\"fields\":[{\"name\":\"title\"},{\"name\":\"summary\"}]}}"

curl "$BASE_URL/api/v1/extract_structured_data/{request_id}"

평가

curl -X POST "$BASE_URL/api/v1/score_extraction_results" \
  -H "content-type: application/json" \
  -d "{\"predicted\":{\"title\":\"A\"},\"reference\":{\"title\":\"A\"},\"rubric_id\":1}"

Workflows

워크플로우는 ocr, marker_parse 같은 step을 저장하고 실행하는 경량 orchestration API입니다.

curl "$BASE_URL/api/v1/workflows/step_types"

curl -X POST "$BASE_URL/api/v1/workflows/workflows" \
  -H "content-type: application/json" \
  -d "{\"name\":\"OCR Workflow\",\"steps\":[{\"step_key\":\"ocr\",\"unique_name\":\"ocr_step\",\"settings\":{\"max_pages\":1}}]}"

curl -X POST "$BASE_URL/api/v1/workflows/workflows/{workflow_id}/execute" \
  -H "content-type: application/json" \
  -d "{\"input_config\":{\"file_urls\":[\"C:/path/sample.pdf\"]}}"

curl "$BASE_URL/api/v1/workflows/executions/{execution_id}"

Collections & Batch

여러 파일을 한 번에 처리하려면 컬렉션을 만든 뒤 batch run을 시작합니다. 현재 batch operation은 convert_document, segment_document, extract_structured_data를 지원합니다.

curl -X POST "$BASE_URL/api/v1/collections" \
  -H "content-type: application/json" \
  -d "{\"name\":\"demo-set\",\"file_ids\":[\"file_1\",\"file_2\"]}"

curl -X POST "$BASE_URL/api/v1/batch_runs" \
  -H "content-type: application/json" \
  -d "{\"collection_id\":1,\"operation\":\"convert_document\",\"params\":{\"output_format\":\"json\"}}"

curl "$BASE_URL/api/v1/batch_runs/{batch_run_id}/results"

Templates & Eval

템플릿과 평가 루브릭은 OCR 결과를 반복 처리하거나 추출 결과를 평가할 때 쓰는 로컬 JSON 기반 관리 API입니다.

curl -X POST "$BASE_URL/api/v1/templates/promote" \
  -H "content-type: application/json" \
  -d "{\"name\":\"보고서 템플릿\",\"kind\":\"form\",\"content\":{\"template_text\":\"제목={{ title }}\"}}"

curl -X POST "$BASE_URL/api/v1/eval_rubrics" \
  -H "content-type: application/json" \
  -d "{\"name\":\"기본 루브릭\",\"weights\":{\"title\":2,\"summary\":1}}"

Images & Downloads

제품 OCR API의 marker 결과는 JSON/Markdown/HTML/chunks를 반환합니다. 이미지 파일까지 포함해 전달해야 할 때는 체험 UI 보조 download endpoint가 ZIP을 생성합니다.

curl -X POST "$BASE_URL/playground/api/convert/start" \
  -F "file=@sample.pdf" \
  -F "page_range=0"

curl "$BASE_URL/playground/api/convert/{request_id}"
curl -o result.zip \
  "$BASE_URL/playground/api/download/{request_id}"

Runtime Settings

자주 바꾸는 timeout, playground 제한, 국회 API 연동 주소는 이미지 재빌드 없이 런타임 설정 파일에 저장할 수 있습니다. 배포 기본 경로는 /data/runtime/runtime-config/settings.json입니다. 설정 변경은 /playground/admin 관리자 페이지 또는 관리자 세션을 가진 API 요청에서만 허용됩니다.

curl -c cookies.txt -X POST "$BASE_URL/playground/api/auth/login" \
  -H "content-type: application/json" \
  -d "{\"username\":\"admin\",\"password\":\"<admin-password>\"}"

curl -b cookies.txt -X PUT "$BASE_URL/playground/api/admin/runtime-settings" \
  -H "content-type: application/json" \
  -d "{\"values\":{\"ocr_service_timeout_sec\":300,\"playground_default_max_pages\":20,\"target_api_timeout_sec\":60}}"

운영 체크리스트

  • OCR_MAX_CONCURRENT_REQUESTS=1은 단일 vLLM/GPU 기준 안전한 기본값입니다.
  • 동시에 2-5명이 써도 요청 결과는 request_id별로 분리되고, 모델 호출은 gate를 통과합니다.
  • 처리량을 늘릴 때는 vLLM --max-num-seqs, GPU 메모리, max pages, page range 제한을 함께 조정합니다.
  • 운영 공개 시 API key, gateway rate limit, upload size limit, request cleanup 정책을 설정합니다.
  • 연동 문서는 /openapi.json과 이 Guide가 같은 구현 상태를 가리키는지 배포 전 확인합니다.
